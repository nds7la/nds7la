# 🚀 GOOD NEWS НА RAILWAY - Гайд для чайника

## ЧТО ТАКОЕ RAILWAY?

Railway - это облачный сервис. Думай как аренда сервера в интернете.
Твой проект будет работать 24/7 в облаке, а ты просто им управляешь.

**Стоимость:** ~$10-20 в месяц (очень дешево)

---

## ЭТАП 1: Подготовка (5 минут)

### 1.1 Создай аккаунт GitHub (бесплатно)

1. Зайди: https://github.com/signup
2. Заполни:
   - Email: твой email
   - Password: придумай пароль
   - Username: придумай юзернейм (например: good-news-user)
3. Подтверди email
4. **Готово, ты в GitHub!**

GitHub нужна чтобы Railway видел твой код.

### 1.2 Загрузи файлы на GitHub

**ПРОЩЕ ВСЕГО - через веб-интерфейс:**

1. Логинишься в GitHub (https://github.com)
2. Нажимаешь "+" в углу → "New repository"
3. Называешь его: `good-news`
4. Нажимаешь "Create repository"
5. Открывается пустой репо

Теперь загрузь файлы:

1. В новом репо нажми "Add file" → "Upload files"
2. Или просто перетащи все файлы в браузер
3. Загрузи все 16 файлов что я отправил
4. Нажми "Commit changes"

**ГОТОВО! Твой код в облаке!** ☁️

---

## ЭТАП 2: Railway Setup (5 минут)

### 2.1 Создай аккаунт Railway

1. Зайди: https://railway.app
2. Нажми "Start Building"
3. Выбери "GitHub" (логинишься через GitHub)
4. Разреши доступ к своему репо
5. **Готово!**

### 2.2 Создай новый проект

1. Нажми "Create new Project"
2. Выбери "Deploy from GitHub"
3. Выбери свой репо `good-news`
4. Railway автоматически обнаружит что это Next.js проект
5. Нажми "Deploy"

Railway начнёт устанавливать и запускать проект автоматически! 

(Это займет 5-10 минут, будут логи в консоли)

---

## ЭТАП 3: Настрой базу данных (3 минуты)

Railway автоматически создаёт PostgreSQL для тебя!

### 3.1 Проверь что база создана

1. В Railway нажми на свой проект
2. Слева ищи "PostgreSQL" в списке сервисов
3. Если её нет - нажми "+" и выбери "PostgreSQL"

### 3.2 Автоматические переменные окружения

Railway АВТОМАТИЧЕСКИ создаёт переменные для базы!

Проверь:
1. На странице проекта → "Variables"
2. Должны быть переменные вроде:
   - `DATABASE_URL` (автоматически)
   - `POSTGRES_HOST`
   - `POSTGRES_PASSWORD`

**Не трогай их, Railway сам всё настроил!**

### 3.3 Добавь недостающие переменные

Railway создал только базу данных. Нужно добавить API ключи.

В "Variables" добавь:

```
NEXTAUTH_SECRET = (придумай любую длинную строку, например: my-super-secret-key-12345)
NEXTAUTH_URL = (это будет автоматически после деплоя, пока можно пропустить)
```

**API ключи добавишь позже** (когда их получишь)

## ЭТАП 4: Дай Railway время на деплой (10 минут)

1. В Railway нажми на свой проект
2. Сверху ищи "Deployments"
3. Должна быть активная сборка (синий статус)
4. Жди пока не станет зелёная галочка "Success"

Это может занять 10-15 минут в первый раз.

**Пока жди, можешь пить кофе! ☕**

---

## ЭТАП 5: Запусти миграции БД (2 минуты)

Когда деплой готов:

1. В Railway нажми на свой проект
2. Нажми "Deployments" → выбери последний
3. Нажми на вкладку "Logs"
4. В консоли внизу напиши:

```
npx prisma migrate deploy
```

Это создаст все таблицы в базе.

Жди пока не закончится и не будет ошибок.

## ЭТАП 7: Добавь API ключи (5 минут)

Теперь нужны ключи для Claude и Replicate.

### 7.1 Claude API ключ

1. Зайди: https://console.anthropic.com/account/keys
2. Нажми "Create Key"
3. Скопируй ключ
4. Вернись в Railway → Variables
5. Добавь переменную:
   ```
   ANTHROPIC_API_KEY = sk-ant-xxxxx (твой ключ)
   ```
6. Сохрани

### 7.2 Replicate API ключ

1. Зайди: https://replicate.com/account/api-tokens
2. Нажми "Create token"
3. Скопируй токен
4. Вернись в Railway → Variables
5. Добавь переменную:
   ```
   REPLICATE_API_TOKEN = r8_xxxxx (твой токен)
   ```
6. Сохрани

### 7.3 Обнови ссылку NEXTAUTH_URL

1. Railway автоматически даст тебе ссылку
2. В Variables добавь:
   ```
   NEXTAUTH_URL = https://good-news-production.up.railway.app
   ```

## ⚠️ ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ НА RAILWAY:

### "Деплой не успешен (красная ошибка)"

1. Нажми на деплой
2. Ищи в логах что пошло не так
3. Обычно проблема в:
   - Неправильном синтаксе файла
   - Недостающей переменной
   - Ошибке в коде

**Решение:** Проверь что все файлы загружены правильно.

### "База не подключается"

1. Railway → Variables
2. Проверь что `DATABASE_URL` есть
3. Если нет - Railway должна создать сама

**Решение:** Перезагрузи приложение (нажми "Restart" в Railway)

### "Страница грузится бесконечно"

1. Жди дольше (первый запуск может быть медленным)
2. Обнови страницу (Ctrl+R)
3. Проверь интернет

### "Логин не работает"

Пароль: `password123` (заглавная P в начале!)

Если не сработало:
1. Railway → Logs
2. Ищи ошибки в логах
3. Обычно проблема в базе данных

**Решение:** Запусти миграции ещё раз (см. ЭТАП 5)
